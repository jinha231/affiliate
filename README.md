# 원더 × 디퍼플 제휴처 관리 대시보드

롯데손해보험 NTC 부업 연계 **제휴 매체를 월별로 관리**하는 단일 HTML 대시보드입니다.
별도 설치·서버·빌드 없이 `affiliate-dashboard.html` 파일 하나로 동작하며, 모든 데이터는
사용자 브라우저의 **localStorage**에 저장됩니다.

---

## 🚀 빠른 시작

1. `affiliate-dashboard.html` 파일을 더블클릭하거나 브라우저로 엽니다.
2. 끝입니다. (인터넷 연결·로그인·설치 불필요)

> **GitHub Pages로 호스팅하려면** 파일명을 `index.html`로 바꿔 저장소에 올리면
> `https://<아이디>.github.io/<레포명>/` 에서 바로 열립니다.

---

## 🧭 메뉴(탭) 구성

| 탭 | 설명 |
|---|---|
| **전체 리스트** | 선택한 월의 전체 제휴처를 표로 관리. 상단에 **월별 컨텍 현황 누적 그래프** 제공 |
| **컨텍 진행 관리** | 컨텍 "중"인 제휴처를 **4단계 파이프라인 매트릭스**로 관리(단계별 메모) |
| **컨텍 종료** | **집행 확정 / 실패**로 종료된 제휴처의 종료 월·세부 메모 관리 |
| **차월 제휴처 📅** | 다음 달 제휴 매체 리스트를 **자동 구성**(이월 + 신규 30개) |

## ✨ 주요 기능

### 1. 컨텍 상태 관리 (4단계)
- `컨텍 예정` → `컨텍 중` → `집행 확정` / `실패`
- 상태별 색상 구분(예정=파랑, 중=주황, 확정=녹색, 실패=적색)
- 상단 통계 카드 5종(전체·예정·중·확정·실패) 실시간 집계

### 2. 월별 누적 그래프
- 월(6/7/8월…)마다 예정·중·확정·실패 **건수를 누적 막대**로 시각화
- 카테고리·검색 필터에 연동

### 3. 컨텍 진행 관리(파이프라인)
- **세로축 = 제휴처 / 가로축 = 컨텍 단계**(① 초기컨택 ② 제휴방향논의중 ③ 적용중 ④ 완료) 매트릭스
- 각 셀(제휴처 × 단계)마다 **독립 메모** 작성, 메모 우상단 **⤢ 버튼**으로 크게 보기/편집
- ④ 완료 단계에서 결과 메모 작성 후 **[집행 확정]/[실패]** 선택 → 컨텍 종료 탭으로 이동

### 4. 컨텍 종료 관리
- 결과 배지(집행 확정/실패) · 종료 월(달력 선택) · 세부 메모

### 5. 차월 제휴처 자동 구성
- **차월 = 현재 월 + 1** (한국시각 기준)
- **트리거 안내**: 매월 *20일이 속한 주의 첫 영업일 09:00(KST)* 도래 시 배너 강조
- **[🔄 리스트 업데이트]** 클릭 시:
  - **(상단) 이월 매체** — 금월 `컨텍 중`·`컨텍 예정` 매체를 차월로 이월(30개 카운트에서 제외)
  - **(하단) 신규 매체 30개** — 이미 컨텍한 매체(확정·실패·예정·중)를 제외한 **미사용 신규 매체**를 내장 후보 풀에서 공급
- **내장 후보 풀 약 109개** 제공(여러 달치). 풀 현황(총/미사용)이 탭에 표시됨
- **+ 후보 매체 추가** 버튼으로 사용자가 신규 후보 매체를 직접 등록 가능 → 풀이 소진돼도 무한 보충(브라우저 저장)
- 재실행 시 기존 차월 리스트를 초기화 후 재구성(확인창 보호)

### 6. 엑셀 다운로드
- 각 탭마다 **⬇ 엑셀 다운로드** 버튼 제공 → 현재 데이터를 `.xls` 파일로 저장
- 파일명: `전체리스트_2026-06.xls`, `컨텍진행관리_…`, `컨텍종료_…`, `차월제휴처_…`

### 7. 제휴처 추가/삭제
- **+ 제휴처 추가** 버튼으로 신규 등록(월 최대 30개)
- 행별 **삭제** 버튼

### 8. 자동 저장 (localStorage)
- 모든 변경(상태·단계·메모·추가·삭제·차월 생성)은 자동 저장되어 새로고침 후에도 유지
- 푸터의 **"초기 데이터로 되돌리기"**로 저장 내용 초기화 가능

## 🗂 데이터 항목

각 제휴처 레코드 필드:

| 필드 | 설명 |
|---|---|
| `month` | 관리 월 (예: `2026-06`) |
| `cat` | 카테고리(프리랜서 플랫폼, 재능마켓, 배달·라이더 등 10종) |
| `access` | 접속 방법(웹 / 앱 / 웹+앱) |
| `name` / `desc` / `target` | 서비스명 / 설명 / 타겟 |
| `email` / `tel` | 대표 메일 / 연락처 |
| `status` | `예정` · `중` · `확정` · `실패` |
| `currentStage` / `stageMemos[]` | 파이프라인 현재 단계 / 단계별 메모 |
| `doneMonth` / `doneMemo` | 종료 월 / 종료 세부 메모 |
| `carried` | 차월 이월 매체 여부 |

## ⚙️ 기술 스택 / 구조

- **순수 HTML + CSS + Vanilla JavaScript** (프레임워크·외부 라이브러리 없음)
- 단일 파일(`affiliate-dashboard.html`)에 마크업·스타일·스크립트·데이터 모두 인라인
- 저장소: 브라우저 `localStorage` (키: `wonder_dpurple_affiliates_v1`)
- 엑셀 내보내기: HTML 테이블 기반 `.xls`(외부 의존성 없음)

## 📌 알아두기 / 제약

- **데이터는 브라우저별로 저장**됩니다. 다른 PC·브라우저와 공유되지 않습니다.
  (여러 사용자 공유가 필요하면 서버 또는 Google Sheets/Firebase 연동이 필요합니다.)
- 백엔드가 없어 차월 트리거는 *정확히 09시에 자동 실행*되는 것이 아니라,
  **해당 시점 이후 처음 앱을 열 때** 도래 안내 + 버튼 갱신 방식입니다.
- 신규 매체는 앱 내장 **후보 풀(약 109개)**에서 공급됩니다. 풀이 소진되면
  차월 탭의 **[+ 후보 매체 추가]** 버튼으로 사용자가 직접 보충할 수 있습니다.
  (GitHub 정적 배포본에서도 동작 — 추가 매체는 해당 브라우저의 localStorage에 저장)
- `.xls` 파일을 Excel로 열 때 "형식과 확장자가 다릅니다" 안내가 뜰 수 있으나,
  **[예]**를 누르면 정상적으로 열립니다(서식·한글 유지).
